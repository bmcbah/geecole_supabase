# Contexte applicatif

Le header permet de sélectionner le workspace, l’établissement et l’année scolaire.

Le contexte est propagé aux pages, services, requêtes, imports et exports. Une action ne peut pas écrire dans un autre établissement ou une autre année sans permission explicite.

## Multi-workspace

Le modèle prévoit plusieurs workspaces. Les règles détaillées de partage, de changement de workspace, de facturation et d’administration globale restent à décrire dans une spécification dédiée avant implémentation complète.

# Bulletins — Règles métier

## Cycle de vie

```text
Non calculable → À compléter → Prêt à générer → Généré → À valider → Validé → Publié
```

## Génération

La génération peut cibler :

- toute l’école ;
- des cycles ;
- des niveaux ;
- des classes ;
- des élèves.

Une erreur sur un élève ne bloque pas les autres élèves prêts.

Un résultat reporté bloque le bulletin de l’élève concerné.

## Validation

Une seule validation est nécessaire.

L’établissement configure plusieurs profils possibles. Un utilisateur appartenant à l’un de ces profils peut valider.

Il n’existe ni circuit multi-étapes ni ordre de validation.

La publication est une action séparée.

## Appréciations

Par défaut, l’enseignant de la matière gère son appréciation. Les profils supplémentaires sont configurables.

Une modification avant publication peut invalider la validation.

## Bulletin figé

Un bulletin publié est un document officiel figé.

Une correction ultérieure :

- ne modifie pas la version publiée ;
- crée une nouvelle version ;
- exige une nouvelle validation ;
- exige une nouvelle publication.

Toutes les versions sont conservées.

## Archivage

Chaque version publiée est archivée dans les documents administratifs de l’élève.

## Structure obligatoire d'une spec

- objectif ;
- contexte et dépendances ;
- acteurs et permissions ;
- règles métier ;
- parcours utilisateur ;
- écrans ;
- modèle de données ;
- sécurité/RLS ;
- événements et intégrations ;
- cas limites et erreurs ;
- critères d'acceptation ;
- questions ouvertes ;
- décisions validées.

Une spec n'est prête au développement que lorsque ses questions bloquantes sont fermées et que son statut passe à `spécifié`.
